title: Plans and scenarios

Plans and scenarios are the foundation of planning in SIMCEL. They allow you to create, compare, and evaluate different versions of your business plan.

## Plans

A **plan** represents a planning cycle with a defined time horizon. Each plan contains one or more scenarios.

### Plan types

| Type | Purpose | Typical horizon |
|------|---------|----------------|
| **Budget** | Annual strategic planning | 12-18 months |
| **Tactical** | Short-term operational adjustments | 3-6 months |

### Creating a plan

When you create a new plan, you define:

1. **Plan type** - Budget or Tactical
2. **Forecast method** - The statistical method used to generate the base forecast
3. **Date range** - The start and end dates for the planning period

<Note>
Once a plan is created, SIMCEL automatically generates a **Forecast Base** scenario using the selected forecast method. This serves as your starting point.
</Note>

### Managing plans

- Switch between plans using the **Plan Bar** in the top navigation
- Only the most recent plan of each type can be deleted
- Each plan maintains its own set of scenarios, events, and simulation results

## Scenarios

A **scenario** is a version of your plan with specific assumptions and events applied. Scenarios let you ask "what if?" questions.

### Scenario types

| Type | Description | Editable? |
|------|-------------|-----------|
| **Forecast Base** | Auto-generated baseline forecast | No |
| **Primary** | Your main working scenario | Yes |
| **Custom** | Additional what-if scenarios | Yes |
| **Actual** | Real performance data imported from your systems | No |

### Scenario flags

Scenarios carry flags that determine their role:

- **PRIMARY** - The main scenario used for reporting and comparison
- **ACTUAL** - Contains actual performance data
- **COMMITTED** - A committed supply plan
- **FORECAST_BASE** - The automatically generated base forecast

### Working with scenarios

#### Creating a scenario
Click the **+** button in the Scenario Bar to create a new scenario. Give it a name and optionally a color for easy identification on charts.

#### Comparing scenarios
Toggle multiple scenarios on using the chips in the Scenario Bar. Charts and tables will display all active scenarios side by side, making it easy to compare outcomes.

#### Scenario colors
Each scenario is assigned a color that is used consistently across all charts and tables. You can customize the color by editing the scenario.

<Tip>
Use scenario comparison to evaluate the impact of different events or assumptions. For example, create one scenario with a promotional event and another without to measure the expected uplift.
</Tip>

## How plans and scenarios work together

```
Plan (Budget 2025)
├── Forecast Base (auto-generated)
├── Primary Scenario (your main plan)
├── Scenario A: "Best Case" (optimistic assumptions)
├── Scenario B: "Conservative" (cautious assumptions)
└── Actual (imported real data)
```

Each scenario within a plan shares the same time horizon and master data but can have different:
- **Events** applied (promotions, market changes, disruptions)
- **Demand adjustments** (manual overrides, shaping)
- **Simulation results** (supply chain outcomes)
- **KPI values** (calculated from the above)

This structure allows you to maintain a single source of truth (the plan) while exploring multiple possible outcomes (scenarios).
